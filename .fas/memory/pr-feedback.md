# Persistent PR Feedback Memory

[2026-06-10] **Markdown outside `.fas/` isn't linted by `verify`/`lint`** (PR #15, CodeRabbit). The repo's `lint:md` (run by `pnpm lint` → `verify.sh`) only covers `.fas/TASKS.md` + `.fas/tasks/**`. Markdown under `docs/` and `packages/**` is only linted by `lint:md:all` (NOT in the verify path) or by CodeRabbit. So `verify.sh --full` can be fully green while new docs still fail markdownlint. **Before pushing any new/edited markdown outside `.fas/`, run `pnpm exec markdownlint-cli2 --config .markdownlint.jsonc <files>`.** Recurring rules to pre-empt: MD040 (every fenced code block needs a language tag — use `text` for ASCII diagrams, `bash` for shell, `ts` for TS) and MD060 (table separator rows must be spaced: `| --- | --- |`, not `|---|---|` — matches the house style in sibling design docs).
