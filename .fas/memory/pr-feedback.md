[2026-06-10] **CLI lifecycle patterns CodeRabbit catches (PR #17)** — three reusable rules for any command-surface code: (1) *Paired execution paths must share one teardown implementation* — the interactive REPL exit skipped the `flush()` that the scripted `--exec` path did; extract shutdown/drain into one function both paths call instead of duplicating steps. (2) *Never split a command string on a bare delimiter when payloads can contain it* — `script.split(';')` broke JSON like `{"text":"a;b"}`; use a quote-aware splitter (track quote state + backslash escapes) for any "commands with JSON/args" surface. (3) *Fire-and-forget promises in process-lifecycle callbacks (`rl.on('close')`, signal handlers) need `.catch` that logs and exits non-zero* — `.then`-only leaves teardown failures as unhandled rejections. Also: `create-task.sh` truncates the brief H1 to the 60-char slug — fix the heading to the full title after creating a brief.
