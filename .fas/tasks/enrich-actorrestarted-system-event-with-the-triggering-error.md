# Enrich actorRestarted system event with the triggering error

## Source

Created with `fas create-task` on 2026-06-13.

## Problem

Follow-up from supervisor-trees (PR #28). actorRestarted carries { address, restartAttempt, supervisor } but not the error that caused the restart; the error is log-only (warn at actor-system-impl.ts ~3832), while give-ups already carry error via actorEscalated. Event-only observers can see that a restart happened but not why. Observability-only.
